# Corpus — Accumulated Thinking

Background context the operator has accumulated over time. Not
rules — those live in `doctrine/`. Not build artifacts — those
live in `builds/`. The corpus is append-only reference material
Sam scans when a conversation's topic maps cleanly to a subdirectory.

| Subdirectory        | Scope                                                       |
|---------------------|-------------------------------------------------------------|
| `concepts/`         | Cross-domain thinking on agents, governance, architecture   |
| `market_structure/` | Settlement, clearing, tokenization, FMI                     |
| `competitive/`      | Firm / product / positioning analysis                       |
| `strategies/`       | Product strategy, market entry, pricing                     |
| `latam/`            | LATAM-specific regulatory, market, client materials         |
| `personal/`         | Sam meta-reflections + operator calibration notes           |

**Not rules.** A corpus entry can be argued with, revised, or
rejected. The operator may disagree with anything here on any
given day. Doctrine is what Sam enforces; corpus is what Sam
consults.

**Append-only.** Sam proposes entries; the operator authorizes
them. Sam does not modify or delete corpus files without explicit
direction.
